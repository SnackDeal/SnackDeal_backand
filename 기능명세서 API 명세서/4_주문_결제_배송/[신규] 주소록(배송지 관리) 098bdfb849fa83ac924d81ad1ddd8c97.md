# 주소록(배송지 관리) (신규)

사용자/관리자 구분: 사용자
연관화면: 마이페이지, 주문정보입력(결제)
도메인: user/delivery
기능설명: 기존 사용자가 배송지(주소록)를 등록·수정·삭제하고 기본 배송지를 지정한다
우선순위: 상
API : /delivery (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/delivery%20ab4bdfb849fa838ca6e481e63c5235dd.md)

### 선행조건

- 로그인 상태여야 한다

### 트리거

- 마이페이지 > 배송 주소록 관리 진입
- 주문정보입력(결제) 화면에서 "배송지 선택/추가"

### 동작

- 회원이 저장한 배송지(delivery) 목록을 표시한다 (기본 배송지 상단 노출)
- 배송지 등록/수정: 배송지명(name), 수령인(receiver_name), 연락처(receiver_phone), 우편번호(zipcode), 주소(address), 상세주소(detail_address)
    - 우편번호/주소는 주소 검색(다음 우편번호 등) 연동
- 기본 배송지(is_default) 지정 — 한 회원당 기본 배송지는 1개만 유지
- 배송지 삭제 (소프트 삭제, deleted_at)
- 주문(결제) 시 주소록에서 배송지를 선택하면 order/prepare의 shipping 값으로 채워진다

### 예외 / 제한사항

- 필수 입력값(수령인, 연락처, 우편번호, 주소) 누락 시 저장 제한
- 기본 배송지로 지정 시 기존 기본 배송지는 자동 해제
- 주문에 사용된 배송지 삭제는 허용하되, 주문의 배송정보는 shipping 테이블에 스냅샷으로 별도 보존 (주소록과 무관)

### 비고

- 배송지는 회원정보(member)와 분리된 delivery 테이블에서 다건 관리한다
- 주문 시점의 배송 정보는 shipping 테이블에 별도 저장되어, 이후 주소록을 수정/삭제해도 과거 주문 배송지는 유지된다
