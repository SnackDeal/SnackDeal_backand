# 주문정보입력 페이지(결제) (수정됨)

사용자/관리자 구분: 사용자
연관화면: 결제
도메인: user/order
기능설명: 기존 사용자가 구매를 하기위해 주문 정보를 입력한다
우선순위: 상
API : /order/prepare (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/order%20prepare%2057ebdfb849fa821b9b4681f37f3ec0f7.md), /order/complete (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/order%20complete%20b57bdfb849fa8289ab1001a6c7d3e8fe.md)

### 선행조건

- 로그인 상태여야 한다
- 장바구니에 1개 이상의 상품이 담겨 있어야 한다

### 트리거

- 장바구니에서 "주문하기" 클릭 / 상품 상세에서 "바로 구매" 클릭

### 동작

- 주문 상품 요약을 표시한다
- 배송지 정보 입력: **주소록(delivery)에서 저장된 배송지를 선택**하거나 직접 입력 (수령인, 연락처, 우편번호, 주소, 상세주소, 배송 요청사항)
- 보유 쿠폰(user_coupon) 목록에서 선택 시 할인 금액을 재계산한다
- 최종 결제 금액 표시 (상품 총액 - 쿠폰 할인 + 배송비)
- 결제 요청 및 승인 처리 (**포트원 SDK → 토스페이먼츠 테스트 결제**)
- 결제 승인 시 주문을 생성하고 재고를 차감한 후 주문완료 페이지로 이동한다

### 예외 / 제한사항

- 결제 실패 (PG 오류, 카드 승인 거절, 잔액 부족 등)
- 주문 시점 재고 부족 시 주문을 차단하고 안내한다
- 필수 배송정보 누락 시 제출을 제한한다
- 선택한 쿠폰이 만료되었거나 최소 주문 금액 조건 미달 시 사용을 차단한다

- 재고관련 사항(동시성 문제가 발생할수 있음)
    - 사용자는 재고 없을 때 미리 안내받아서 결제창까지 안 감 ✓
    - 결제 실패해도 되돌릴 재고가 없음 (복구 로직 불필요) ✓
    - 코드가 단순해서 버그도 적음 ✓

```jsx
1. 사용자가 "결제하기" 클릭
2. 재고 확인 (SELECT만, 차감 X)
   → 부족하면 즉시 "품절" 안내, PG 호출도 안 함
   → 충분하면 다음 단계
3. PG 결제 요청
4. PG 승인 성공 응답 받음
5. 트랜잭션으로 재고 재확인 + 차감 + 주문 생성 (원자적으로)
```

결제 승인 시점에 재고 차감할 때, **DB 트랜잭션으로 재고 확인 + 차감을 원자적으로 처리예시**

```jsx
BEGIN
  SELECT stock FROM products WHERE id = ? FOR UPDATE  -- 락
  IF stock >= 주문수량 THEN
    UPDATE products SET stock = stock - 주문수량 WHERE id = ?
    INSERT INTO orders ...
    COMMIT
  ELSE
    ROLLBACK -> 결제 취소 처리
  END IF
END
```
