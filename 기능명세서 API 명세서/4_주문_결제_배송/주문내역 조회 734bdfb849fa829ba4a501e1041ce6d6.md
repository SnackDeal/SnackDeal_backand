# 주문내역 조회

사용자/관리자 구분: 사용자
연관화면: 마이페이지, 상단페이지
도메인: user/mypage
기능설명: 기존 사용자가 주문한 주문 내역을 확인한다
우선순위: 상
API : /order/list (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/order%20list%20769bdfb849fa83f8bff08148f42f9844.md), /order/{order_id} (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/order%20%7Border_id%7D%20ff7bdfb849fa835b911f01daa91f12b9.md), /order/{order_id}/refund (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/order%20%7Border_id%7D%20refund%20b9fbdfb849fa8212b6a981598a6d9069.md)

### 선행조건

- 로그인 상태여야 한다

### 트리거

- 마이페이지 > 주문내역 진입

### 동작

[마이페이지 - 주문내역 조회](https://app.notion.com/p/390bdfb849fa800b92add1f7d045c533?pvs=21) 

- 본인의 주문 목록을 최신순으로 표시한다
- 검색
    - 주문처리상태(결제완료/배송준비중/배송중/배송완료/취소)
    - 오늘/1개월/3개월/6개월
    - 날짜 검색(1년)

- 목록 항목: 주문일자, 상품명, 주문 금액, 주문 상태(결제완료 / 배송중 / 배송완료 / 취소)
- 주문 상세로 이동하여 상품별 내역, 배송지, 결제수단, 결제금액을 확인할 수 있다

#### 6. 하단 액션 영역

주문 상태에 따라 다르게 노출:

| 상태 | 노출되는 버튼 |
| --- | --- |
| 결제완료 | 환불 요청 |
| 배송준비중 | 환불 요청 |
| 배송중 | (환불 안내: 고객센터 문의) |
| 배송완료 | (환불 안내: 고객센터 문의) |
| 환불요청 | 처리 대기 안내 |
| 환불완료 / 취소 | 액션 없음 |

### 예외 / 제한사항

- 주문 내역이 없을 경우 안내 문구를 표시한다
- 타인의 주문 상세에 접근할 수 없다

### 비고

- 주문 취소/환불은 사용자가 누르고 나서 관리자에서 승인 후 처리함
- 기타 배송중 이후 상태일 경우 고객센터 문의로 ….
