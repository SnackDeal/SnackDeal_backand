## GET /delivery — 내 배송지 목록

메서드: GET

### Response `200`

```json
{
  "deliveries": [
    {
      "id": 5,
      "name": "우리집",
      "receiver_name": "홍길동",
      "receiver_phone": "01012345678",
      "zipcode": "06133",
      "address": "서울 강남구 테헤란로 123",
      "detail_address": "456호",
      "is_default": true
    }
  ]
}
```

`401` 미인증

---

## POST /delivery — 배송지 추가

메서드: POST

### Request Body

| 파라미터 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `name` | string | Y | 배송지명 (예: 우리집, 회사) |
| `receiver_name` | string | Y | 수령인 |
| `receiver_phone` | string | Y | 연락처 (하이픈 없이) |
| `zipcode` | string | Y | 우편번호 |
| `address` | string | Y | 기본 주소 |
| `detail_address` | string | N | 상세 주소 |
| `is_default` | bool | N | 기본 배송지 지정 (기본 false) |

```json
{
  "name": "우리집",
  "receiver_name": "홍길동",
  "receiver_phone": "01012345678",
  "zipcode": "06133",
  "address": "서울 강남구 테헤란로 123",
  "detail_address": "456호",
  "is_default": true
}
```

### Response `201`

```json
{ "id": 5, "is_default": true }
```

`400` 필수값 누락 · `401` 미인증

> `is_default=true`로 등록/수정하면 기존 기본 배송지는 자동으로 `false`로 해제된다.

---

## PUT /delivery/{id} — 배송지 수정

메서드: PUT · Body는 POST와 동일. `200` 수정된 배송지 반환 · `403` 타인 배송지 · `404` 없음

---

## PATCH /delivery/{id}/default — 기본 배송지 지정

메서드: PATCH · 해당 배송지를 기본으로 설정하고 기존 기본은 해제. `200` 성공

---

## DELETE /delivery/{id} — 배송지 삭제

메서드: DELETE · 소프트 삭제(deleted_at 기록). `200` 성공 · `403` 타인 배송지 · `404` 없음
