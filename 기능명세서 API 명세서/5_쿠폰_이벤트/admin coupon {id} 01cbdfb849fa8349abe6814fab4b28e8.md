# /admin/coupon/{id}

개발순서: 2차
기능: 관리자_쿠폰관리페이지 (../%EA%B8%B0%EB%8A%A5%EC%A0%95%EC%9D%98%EC%84%9C/%EA%B4%80%EB%A6%AC%EC%9E%90_%EC%BF%A0%ED%8F%B0%EA%B4%80%EB%A6%AC%ED%8E%98%EC%9D%B4%EC%A7%80%2088cbdfb849fa835cb23001d2fd345068.md)
기능설명: 쿠폰 수정
도메인: admin/coupon
메서드: PUT
사용자/관리자: 관리자

### Request

---

#### **Request Body** (수정 가능한 필드만)

| 파라미터 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `name` | string | N | 쿠폰명 |
| `valid_until` | datetime | N | 유효기간 종료일 (연장만 가능, 단축 불가) |
| `total_quantity` | int | N | 발급 수량 (증가만 가능, 감소 불가) |

> 게시일(이벤트게시판 노출 시작)은 coupon이 아니라 소속 **coupon_board.start_at** 을 수정한다. 할인율/할인방식/발급방식(issue_type)/소속 게시판은 발급 후 변경 불가.

### Response

---

`200` 수정 성공 → 최신 쿠폰 정보 반환

`400` 유효성 실패

```json
{ "error": { "code": "BAD_REQUEST", "message": "유효기간 종료일은 단축할 수 없습니다" } }
```

`422` 수정 불가 필드 시도 (할인율, 할인 방식, 발급 방식 등)

```json
{ "error": { "code": "UNPROCESSABLE", "message": "이미 발급된 쿠폰의 할인 조건은 변경할 수 없습니다" } 
```

**왜 이렇게 제한적인가**

- 쿠폰을 받은 사용자가 받았을 때의 조건이 나중에 바뀌면 신뢰 문제 발생
- 예: 5000원 할인 쿠폰을 받았는데 관리자가 나중에 1000원 할인으로 바꿔버리면?
- **연장/증가는 사용자에게 이득**이므로 허용, **단축/감소는 손해**라 차단

`404` 존재하지 않는 쿠폰 · `403` 관리자 권한 없음
