# 관리자_쿠폰관리페이지 (수정됨)

사용자/관리자 구분: 관리자
연관화면: 쿠폰 발급 페이지
도메인: admin/coupon
기능설명: 기존 관리자가 쿠폰 관리 페이지에 진입한다
우선순위: 상
API : /admin/coupon (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/admin%20coupon%203e9bdfb849fa82449aeb01b294db33fd.md), /admin/coupon (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/admin%20coupon%20978bdfb849fa8312a55001ba08409e1b.md), /admin/coupon/{id} (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/admin%20coupon%20%7Bid%7D%2001cbdfb849fa8349abe6814fab4b28e8.md), /admin/coupon/{id}/status (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/admin%20coupon%20%7Bid%7D%20status%20594bdfb849fa823fbdcb017c0a6e0928.md)

### 선행조건

- 관리자 로그인 상태여야 한다

### 트리거

- 쿠폰관리 메뉴 진입 / "신규 쿠폰 등록" 버튼 클릭

### 동작

[관리자 - 쿠폰 관리](https://app.notion.com/p/390bdfb849fa806a886fca9a41cc1ebc?pvs=21) 

**[선행] EVENT 쿠폰은 이벤트게시판(coupon_board) 먼저 등록** — coupon_board_id는 nullable FK다. **EVENT(할인·이벤트) 쿠폰**은 먼저 이벤트게시판을 게시(제목/내용/썸네일/게시기간 start_at~end_at)한 뒤 그 게시판에 등록한다. **SIGNIN(회원가입) 쿠폰**은 게시판 없이(coupon_board_id=null) 등록 가능하다.

- 쿠폰 등록 (coupon)
    - 쿠폰명
    - 소속 이벤트게시판 선택 (coupon_board_id — EVENT 필수 / SIGNIN 생략 가능)
    - 할인 방식(discount_type): 정액 FIXED(원) / 정률 PERCENT(%)
    - 할인 값(discount_value), 최소 주문 금액(min_order_price, 기본 0)
    - 유효기간: 다운로드·사용 시작 지정일(valid_from) ~ 종료(valid_until)
    - 총 발급 수량(total_quantity)
    - 발급 타입(issue_type, 필수):
        - **SIGNIN(회원가입)**: 신규 회원가입 시 자동 지급
        - **EVENT(할인·이벤트)**: 사용자가 이벤트게시판에서 valid_from(지정일)부터 직접 다운로드
    - 강제 활성/중지 플래그(is_active)
- 쿠폰 리스트
    - 컬럼: 쿠폰명, 소속 게시판, 할인 방식/값, 발급 타입(SIGNIN/EVENT), 유효기간, 발급 수량(issued_quantity/total_quantity), 사용 수, 상태(ACTIVE/EXPIRED/STOPPED)
    - 필터: 발급 타입 필터, 상태 필터
- 등록된 쿠폰의 사용중지(is_active=false) 처리 및 수정 (이미 발급된 쿠폰의 할인 조건·발급 타입·소속 게시판은 변경 불가)

### 예외 / 제한사항

- 필수 입력값 누락 (특히 coupon_board_id)
- 소속 게시판(coupon_board)이 존재하지 않음
- 종료일(valid_until)이 시작일(valid_from)보다 이른 경우
- "SIGNIN(회원가입)" 타입 쿠폰은 동시에 여러 개를 활성 상태로 두지 않도록 경고 (중복 지급 방지)

### 비고

- 발급 타입(issue_type)은 등록 후 변경 불가 (쿠폰함/이벤트게시판 노출 정책과 일관성 유지)
- 별도의 "쿠폰 코드 수동 입력" 방식은 없음 (타입은 회원가입/할인 2종)
- 게시일(게시판 노출 시작)은 coupon_board.start_at, 다운로드 지정일은 coupon.valid_from 으로 분리 관리
