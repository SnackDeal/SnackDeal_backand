# 이벤트페이지(쿠폰 다운로드) (수정됨)

사용자/관리자 구분: 사용자
연관화면: 이벤트
도메인: user/event
기능설명: 기존 사용자가 이벤트 페이지에 진입한다
우선순위: 상
API : /event/coupon/{coupon_id}/download (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/event%20coupon%20%7Bcoupon_id%7D%20download%20633bdfb849fa8324b653010c2e3b6c8a.md), /event/coupon/list (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/event%20coupon%20list%20a5dbdfb849fa8396b67281d97d80afe0.md)

### 선행조건

- 회원x

### 트리거

- 이벤트 메뉴 진입

### 동작

- 관리자가 게시한 **이벤트게시판(coupon_board)** 글과 그에 소속된 EVENT 타입 쿠폰 목록을 표시한다
    - 게시판 항목: 제목(title), 내용(content), 썸네일(thumbnail_url), 게시기간(start_at~end_at)
    - 쿠폰 항목: 쿠폰명, 할인 방식/값(discount_type/value), 사용 조건(min_order_price), 유효기간(valid_from~valid_until), 남은 발급 수량(total_quantity - issued_quantity)
    - **게시판은 게시기간(start_at)부터 사전 노출**되며, 쿠폰 "받기"는 **다운로드 지정일(valid_from)부터** 가능하다
- "쿠폰 받기" 버튼 클릭 시 사용자 쿠폰함(user_coupon)에 즉시 지급한다 (status=ACTIVE)
- 이미 받은 쿠폰은 "발급 완료" 상태로 표시하고 재발급을 차단한다

### 예외 / 제한사항

- 비로그인 상태에서 "쿠폰 받기" 클릭 시 로그인 페이지로 이동한다
- 다운로드 지정일(valid_from) 이전에는 "오픈 예정(upcoming)"으로 표시하고 받기 차단한다
- 발급 수량 소진(issued_quantity ≥ total_quantity) 시 "소진(soldout)" 상태로 표시하고 발급을 차단한다
- 유효기간(valid_until)이 지났거나 is_active=false인 쿠폰은 받기 차단한다
- 사용자 1인당 발급 횟수 정책: 쿠폰당 1회 발급 (user_coupon의 member_id+coupon_id 중복 차단)

### 비고

- 이벤트페이지는 게시판(coupon_board) 형태 (ui적인건 관리자에서 처리)
