# 회원가입 (수정됨)

사용자/관리자 구분: 사용자
연관화면: 로그인
도메인: user/member
기능설명: 신규 사용자가 회원가입을 수행한다
우선순위: 상
API :  /member/join (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/member%20join%20995bdfb849fa82669e2381ebc716df41.md)

### 선행조건

- 비회원 상태여야 한다

### 트리거

- 회원가입 폼 제출

### 동작

순서 : 이용약관 및 개인정보처리방침 동의 → 인증 → 회원가입 진행

- 본인 인증은 **이메일 인증**으로 구현한다 (인증코드 발송 → 검증 → 토큰 발급)
- 입력 항목: 이메일, 비밀번호, 비밀번호 확인, 이름, 휴대폰번호(phone), 생년월일(birth, YYYY-MM-DD), 성별(gender: MALE/FEMALE)
- 이용약관 및 개인정보처리방침 동의 필수(cafe24거 그대로복사해서 db에 저장함)
- 유효성 검증을 수행한다
    - 이메일 형식 검증 및 중복 확인(중복확인 버튼)
    - 비밀번호 규칙 검증 (8자 이상, 영문+숫자+특수문자 조합)
    - 비밀번호 확인 일치 여부
- 회원가입 버튼 클릭시 이메일 인증을 진행하고 인증번호가 맞을 시 검증을 통과한다
- 검증 통과 시 계정을 생성하고 가입 완료 처리한다
- 가입 완료 시 "회원가입 쿠폰"(관리자가 회원가입 자동 발급으로 설정한 활성 쿠폰)을 자동으로 사용자 쿠폰함에 지급한다

### 구글(소셜) 회원가입 (`is_social_login=true`)

- 구글 로그인 후 신규 이메일이면 이메일/이름을 들고 회원가입 폼으로 이동하며, 나머지 입력 항목(휴대폰번호·생년월일·성별)만 추가로 입력받는다
- 이메일은 구글 인증으로 이미 확인된 것으로 간주하여 **이메일 인증코드 발송/검증 절차를 생략**한다 (`verification_token` 불필요)
- 비밀번호는 사용자가 입력하지 않으며, 서버가 임의의 비밀번호를 생성해 저장한다 (사용자에게 노출되지 않음, 로그인에 사용하지 않음)
- 가입 완료 시 별도의 `/member/login` 호출 없이 **가입 응답에 access/refresh 토큰을 함께 발급**하여 바로 로그인 세션을 시작한다

### 예외 / 제한사항

- 이메일 중복확인
- 필수 입력값 누락
- 비밀번호 규칙 미충족 또는 확인 불일치 (일반 회원가입만 해당, 소셜 회원가입은 비밀번호 입력 없음)
- 이메일 인증 번호 불일치 (일반 회원가입만 해당)
- 이용약관 미동의

선택) 회원가입 쿠폰 지급 실패 시 회원가입 자체는 성공 처리하고, 지급 실패만 로그로 남긴다

### 비고

회원가입 쿠폰은 관리자 쿠폰 관리 화면에서 "회원가입 자동 발급"으로 설정된 쿠폰이 대상
