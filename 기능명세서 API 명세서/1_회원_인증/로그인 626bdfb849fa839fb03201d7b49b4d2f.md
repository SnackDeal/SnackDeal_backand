# 로그인

사용자/관리자 구분: 사용자
연관화면: 로그인
도메인: user/member
기능설명: 기존 사용자가 인증을 수행한다
우선순위: 상
API : /member/login (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/member%20login%20719bdfb849fa832e92b581ad4f30a51a.md), /member/token/refresh (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/member%20token%20refresh%20455bdfb849fa825ba1d80121cc48c750.md),  /member/logout (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/member%20logout%20accbdfb849fa8210ae6b015a6530369e.md)

### 선행조건

- 등록된 계정이어야 한다

### 트리거

- 로그인 폼 제출(이메일 + 비번)

### 동작

- 입력값 유효성 검증 (필수 입력, 이메일 형식)
- 인증 성공 시 사용자 세션을 생성한다
- 로그인 성공 후 이전 페이지 또는 메인 페이지로 이동한다
- 인증 실패 시 오류 메시지를 표시한다

### 예외 / 제한사항

- 아이디 또는 비밀번호 불일치
- 탈퇴 처리된 계정
- 필수 입력값 누락

### 비고

- 구글 로그인, 일반 로그인 인증 포함

### Google OAuth2 로그인 처리

- **기존 회원(이미 가입된 이메일)**: 구글 인증 성공 시 `GoogleOAuth2MemberService`가 이메일로 회원을 찾아 바로 access/refresh 토큰을 발급하고, 프론트 `/oauth2/callback?access_token=...&refresh_token=...`로 리다이렉트한다. `/member/login`을 별도로 호출하지 않는다.
- **신규 회원(가입되지 않은 이메일)**: 토큰을 발급하지 않고 프론트 `/signup?email=...&name=...`으로 리다이렉트한다. 사용자는 나머지 항목(휴대폰번호·생년월일·성별)을 입력해 `POST /member/join`을 `is_social_login=true`로 호출한다.
    - 이메일 인증 절차(`verification_token`)는 생략된다.
    - 비밀번호는 사용자가 입력하지 않고, 백엔드가 임의의 강력한 비밀번호를 생성해 저장한다(사용자에게 노출되지 않음, 이후 인증에 사용하지 않음).
    - 가입 성공 응답에 access/refresh 토큰이 함께 발급되어, 소셜 회원가입 사용자는 `/member/login`을 별도로 호출할 필요가 없다.
