# 이메일인증 (수정됨)

사용자/관리자 구분: 사용자
연관화면: 회원가입
도메인: user/member
기능설명: 신규 사용자가 회원가입 시 메일로 인증번호를 보내주고 확인한다
우선순위: 상
API : /member/email/send-code (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/member%20email%20send-code%20ee9bdfb849fa83f2825e0131fc8327c6.md), /member/email/verify-code  (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/member%20email%20verify-code%207adbdfb849fa82fd81c88162fc497873.md)

### 이메일 인증 — 인증코드 발송 (사용자)

**선행조건**

- 없음 (비회원 상태)

**트리거**

- 회원가입 폼에서 이메일 입력 후 "인증코드 전송" 버튼 클릭
- 이전 인증코드 만료 후 "재전송" 버튼 클릭

**동작**

- 입력된 이메일의 형식을 검증한다 (`xxx@xxx.xxx` 형태)
- 해당 이메일로 이미 가입된 계정이 있는지 조회한다
- 서버에서 6자리 숫자 인증코드를 랜덤 생성한다
- 인증코드를 DB(또는 Redis)에 저장한다
    - 저장 항목: 이메일, 인증코드, 만료시각(5분 후)
    - 동일 이메일의 기존 미사용 인증코드가 있으면 무효화(덮어쓰기)
- SMTP를 통해 사용자 이메일로 인증코드를 발송한다
- 사용자 화면에는 "인증코드가 발송되었습니다" 안내와 함께 인증코드 입력 필드를 노출한다
- 인증코드 입력 필드 옆에 남은 유효시간(카운트다운)을 표시한다
- 60초 동안 재전송 버튼을 비활성화한다

**예외 / 제한사항**

- 이메일 형식 오류 시 발송을 차단한다
- 이미 가입된 이메일에 대한 요청은 차단한다 ("이미 가입된 이메일입니다")
- 최근 60초 이내 동일 이메일로 재발송 요청 시 차단한다 (스팸 방지)
- SMTP 발송 실패 시 오류 메시지를 표시하고 재시도를 유도한다
- 인증코드 유효시간(5분) 만료 시 사용자에게 재전송을 유도한다

**비고**

- 이메일 서비스: Gmail SMTP 사용 (앱 비밀번호 발급 필요)
- 인증코드는 6자리 숫자 (100000~999999 범위)
- 인증코드 저장 시 대소문자 없이 문자열로 통일
- 이메일 주소는 소문자로 정규화하여 저장·비교
- 개발 환경에서는 실제 발송 없이 콘솔 로그로 대체 가능하도록 설정 분기

### 이메일 인증 — 인증코드 검증 (사용자)

**선행조건**

- 인증코드 발송 요청이 선행되어 있어야 한다

**트리거**

- 사용자가 인증코드 입력 후 "확인" 버튼 클릭

**동작**

- 입력된 이메일과 인증코드를 DB에서 조회한다
- 인증코드 일치 여부와 만료시각을 검증한다
- 검증 성공 시:
    - 사용된 인증코드를 삭제(또는 사용됨 처리)한다
    - 회원가입에 사용할 `verification_token`을 발급한다 (유효시간 10분)
    - 토큰을 DB(또는 Redis)에 저장한다 (이메일과 매핑)
    - 프론트에 토큰을 전달하고 인증 완료 상태를 표시한다
- 검증 완료 후 회원가입 폼의 남은 필드(비밀번호/이름/생년월일/성별 등) 입력을 활성화한다

**예외 / 제한사항**

- 인증코드 불일치 시 오류 메시지를 표시한다
- 인증코드 만료(5분 경과) 시 재전송을 유도한다
- 인증코드 검증 실패 5회 초과 시 해당 인증코드를 무효화하고 재전송을 요구한다 (선택)
- 이미 사용된 인증코드로 재검증 시도 시 차단한다

**비고**

- 인증 성공 후 발급되는 `verification_token`은 1회성 (회원가입 성공 시 삭제)
- 토큰 유효기간(10분)은 사용자가 인증 후 회원가입 폼을 마저 작성하는 시간 확보 목적
- 토큰과 이메일이 매핑되어 있어, 회원가입 시 다른 이메일로 바꿔치기 하는 것을 방지

### 이메일 인증 — 회원가입 시 토큰 검증 (백엔드 내부 처리)

**선행조건**

- 이메일 인증 검증이 완료되어 유효한 `verification_token`이 있어야 한다

**트리거**

- 회원가입 API(`POST /member/join`) 요청 수신

**동작**

- 요청 Body에서 `email`, `verification_token`을 추출한다
- 저장된 토큰과 매핑된 이메일이 요청의 이메일과 일치하는지 검증한다
- 토큰 유효기간(10분) 이내인지 확인한다
- 검증 통과 시 계정을 생성하고 사용된 토큰을 삭제한다
- 검증 실패 시 회원가입을 거부한다

**예외 / 제한사항**

- 토큰 누락 시 `401 UNAUTHORIZED` 반환 ("이메일 인증이 필요합니다")
- 토큰 만료 시 재인증을 요구한다
- 토큰-이메일 불일치 시 요청을 차단한다 (위·변조 시도로 간주)
- 이미 사용된 토큰으로 재요청 시 차단한다

**비고**

- 프론트 측 인증 상태만 신뢰하지 않고, 서버가 반드시 토큰을 재검증한다
- 검증 로직은 회원가입 트랜잭션 내부에서 처리 (계정 생성 성공 시 토큰 삭제까지 원자적으로)

### 개발 시 체크리스트

**보안 관련**

- 인증코드는 반드시 서버에서 생성 (프론트에서 생성하지 않음)
- 인증코드 저장 시 이메일과 함께 저장하되, 인증 성공 시 즉시 삭제
- `verification_token`은 UUID 또는 랜덤 문자열 (예측 불가능하게)
- 토큰과 이메일 매핑을 서버에서 관리 → 이메일 바꿔치기 방지

**UX 관련**

- 인증코드 입력창 옆에 남은 시간 카운트다운 표시
- 60초 재전송 제한 동안 버튼 비활성화 + 남은 시간 안내
- 만료된 코드 사용 시 "재전송" 버튼으로 자연스럽게 유도

**개발 환경 팁**

- Gmail SMTP는 2단계 인증 필수, "앱 비밀번호" 별도 발급
- 개발 중에는 실제 발송 대신 콘솔에 코드 출력하는 옵션 두면 편함
- 환경변수: `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD`
- `.env` 파일 반드시 `.gitignore`에 추가
