# /admin/members/{id}

개발순서: 3차
기능: 관리자_회원 관리 (../%EA%B8%B0%EB%8A%A5%EC%A0%95%EC%9D%98%EC%84%9C/%EA%B4%80%EB%A6%AC%EC%9E%90_%ED%9A%8C%EC%9B%90%20%EA%B4%80%EB%A6%AC%20be0bdfb849fa83b2880a01649d3450d3.md)

도메인: admin/users
메서드: GET
사용자/관리자: 관리자

### Request

---

**Path Params**

| 파라미터 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `id` | long | Y | 회원 ID |

## Response

---

**`200`** 기본 정보 + (선택)누적 주문/결제 + 최근 주문

```json
{
  "id": 45,
  "email": "hong@test.com",
  "name": "홍길동",
  "phone": "01011112222",
  "birth": "2000-01-01",
  "gender": "MALE",
  "status": "ACTIVE",
  "role": "USER",
  "created_at": "2026-06-12T10:30:00Z",
  "last_login": "2026-07-01T14:32:00Z",
  "stats": {
    "total_order_count": 7,
    "total_payment": 143000,
    "available_coupon_count": 2,
    "pending_qna_count": 1
  },
  "recent_orders": [
    {
      "order_id": 123,
      "order_number": "ORD-20260701-00123",
      "ordered_at": "2026-07-01T14:32:00Z",
      "final_amount": 12200,
      "status": "PREPARING_SHIPMENT"
    },
    {
      "order_id": 98,
      "order_number": "ORD-20260628-00098",
      "ordered_at": "2026-06-28T09:15:00Z",
      "final_amount": 3200,
      "status": "COMPLETED"
    }
  ]
}
```

**필드 설명**

- `stats.total_order_count`: 취소·환불 제외한 유효 주문 수
- `stats.total_payment`: 유효 주문의 결제 금액 합계
- `stats.available_coupon_count`: 사용 가능한 쿠폰 수
- `stats.pending_qna_count`: 답변 대기중인 문의 수 (관리자가 우선 처리 판단용)
- `recent_orders`: 최근 5건 (최신순)

`stats`는 매번 계산하지 말고 캐시 또는 집계 쿼리로 처리 권장. MVP에선 매번 count해도 충분.

**개발 부담 줄이려면**

- `stats`와 `recent_orders`를 빼고 기본 정보만 반환하는 최소 스펙도 가능
- 그럴 경우 관리자 회원 상세는 리스트에서 상태 변경만 하는 화면과 다를 게 없어짐

`404` 존재하지 않는 회원 · `403` 관리자 권한 없음
