# 5. 쿠폰 · 이벤트게시판 도메인

쿠폰과 이벤트게시판. **구조: 이벤트게시판(글) → 쿠폰(혜택) → 쿠폰함(회원 보유)** 3단.

## 관련 DB 테이블
- `coupon_board` (이벤트게시판: title, content, thumbnail_url, is_active, start_at, end_at) — 게시(노출) 기간
- `coupon` (name, discount_type FIXED/PERCENT, discount_value, min_order_price, valid_from, valid_until, total_quantity, issued_quantity, issue_type EVENT/SIGNIN, is_active, coupon_board_id **nullable**)
- `user_coupon` (member_id, coupon_id, status ACTIVE/USED/EXPIRED, issued_at, used_at)

## 핵심 개념 (헷갈리기 쉬움)
- **게시일 = coupon_board.start_at** : 이벤트가 화면에 보이기 시작
- **받기 오픈일 = coupon.valid_from** : 실제 다운로드가 열림
- **issue_type**: `EVENT`(게시판 필수, 이벤트페이지에서 다운로드) / `SIGNIN`(회원가입 자동, 게시판 null 가능)
- 관리자 순서: ① 이벤트게시판 등록 → ② 그 게시판 선택해 쿠폰 등록. 쿠폰 코드 입력 방식 없음.

## 화면 (기능정의서)
- **쿠폰함** (사용자) — 보유 쿠폰 ACTIVE/USED/EXPIRED
- **이벤트페이지(쿠폰 다운로드)** (사용자) — 게시판+쿠폰 노출, valid_from부터 받기
- **관리자_쿠폰관리페이지** — 게시판 먼저 등록 후 쿠폰 등록/중지

## API
| 메서드 | 경로 | 설명 |
| --- | --- | --- |
| GET | /mypage/coupon | 내 쿠폰함 (user_coupon) |
| GET | /event/coupon/list | 이벤트 쿠폰 목록(게시판 포함) |
| POST | /event/coupon/{coupon_id}/download | 쿠폰 받기 → user_coupon |
| GET | /admin/coupon | (관리자) 쿠폰 리스트 |
| POST | /admin/coupon | (관리자) 쿠폰 등록 (coupon_board_id: EVENT 필수/SIGNIN null) |
| PUT | /admin/coupon/{id} | (관리자) 쿠폰 수정 |
| PATCH | /admin/coupon/{id}/status | (관리자) is_active 토글 |
| GET/POST/PUT/DELETE | /admin/coupon-board | (관리자) 이벤트게시판 CRUD [신규] |
