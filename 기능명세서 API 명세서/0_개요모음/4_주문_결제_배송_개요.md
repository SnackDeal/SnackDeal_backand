# 4. 주문 · 결제 · 배송 도메인

주문/결제(포트원→토스 테스트), 주문내역/환불, 배송, 그리고 **주소록**. 주문은 3개 테이블로 분리됩니다.

## 관련 DB 테이블
- `orders` (order_number UNIQUE, product_amount, shipping_fee, discount_amount, final_amount, status, member_id, user_coupon_id, cancelled_at …)
  - status: PENDING_PAYMENT / PAYMENT_COMPLETED / PREPARING_SHIPMENT / SHIPPED / COMPLETED / CANCELLED / REFUND_REQUESTED / REFUND_COMPLETED
- `order_item` (product_name 스냅샷, price, quantity, product_id, order_id)
- `shipping` (주문별 배송 스냅샷: receiver_name/phone, zipcode, address, detail_address, delivery_request, courier, tracking_number, status READY/PREPARING/SHIPPING/DELIVERED) — order_id UNIQUE
- `payment` (imp_uid, merchant_uid, amount, pay_method, pg_provider=tosspayments, status READY/PAID/FAILED/CANCELLED, receipt_url) — order_id UNIQUE
- `delivery` (주소록: name, receiver_name/phone, zipcode, address, detail_address, is_default, member_id) — 재사용 저장 배송지

## 화면 (기능정의서)
- **주문정보입력 페이지(결제)** — 배송지(주소록 선택/직접입력)+쿠폰→포트원 토스 테스트 결제. prepare→SDK→complete 재검증
- **주문내역 조회** — 내 주문 목록/상세/환불 요청
- **주소록(배송지 관리)** [신규] — 배송지 CRUD, 기본 배송지 지정
- **관리자_주문관리** — 주문 목록/상세/상태변경/환불 처리

## API
| 메서드 | 경로 | 설명 |
| --- | --- | --- |
| POST | /order/prepare | 주문 임시 생성+금액 확정 (결제창 전 재고 체크) |
| POST | /order/complete | 결제 검증+주문 확정(재고 차감·쿠폰 사용) |
| GET | /order/list | 내 주문내역 |
| GET | /order/{order_id} | 주문 상세 |
| POST | /order/{order_id}/refund | 환불 요청 → REFUND_REQUESTED |
| GET | /delivery | 주소록 목록 |
| POST | /delivery | 배송지 추가 |
| PUT | /delivery/{id} | 배송지 수정 |
| PATCH | /delivery/{id}/default | 기본 배송지 지정 |
| DELETE | /delivery/{id} | 배송지 삭제 |
| GET | /admin/order | (관리자) 주문 리스트 |
| GET | /admin/order/{id} | (관리자) 주문 상세 |
| PATCH | /admin/order/{id}/status | (관리자) 배송 상태 변경 |
| POST | /admin/order/{id}/refund | (관리자) 환불 승인→REFUND_COMPLETED / 거절 |

> delivery(주소록)와 order API는 원래 별도 폴더였고, 신규 delivery API 파일은 옛 "제목 없음 ab4…" 파일을 재활용한 것입니다.
