# 2. 상품 · 카테고리 도메인

상품 조회(사용자)와 상품/카테고리 관리(관리자). **옵션 없음, 상품 이미지 1장**이 핵심 규칙.

## 관련 DB 테이블
- `product` (id, name, price, description, status ACTIVE/INACTIVE/DELETED, stock, category_id …) — 옵션 컬럼 없음
- `product_image` (attachment_url, sort_order, product_id) — 상품당 1행(대표 이미지)
- `category` (id, name UNIQUE, sort_order, deleted_at)

## 화면 (기능정의서)
- **상품리스트페이지** (사용자) — 카테고리/검색/정렬/페이지네이션, 품절 배지
- **상품 상세 페이지** (사용자) — 대표 이미지 1장·가격·설명·재고, 장바구니/바로구매 (옵션 없음, 수량만)
- **관리자_상품관리 리스트(재고 포함)** — 재고/상태 필터
- **관리자_상품 등록 / 수정** — 상품 등록·수정, 재고 통합 관리

## API
| 메서드 | 경로 | 설명 |
| --- | --- | --- |
| GET | /product/list | 상품 리스트 |
| GET | /product/{product_id} | 상품 상세 (image_url 단일) |
| GET | /admin/product | (관리자) 상품 리스트(재고) |
| POST | /admin/product | (관리자) 상품 등록 |
| GET | /admin/product/{id} | (관리자) 상품 상세(수정용) |
| PUT | /admin/product/{id} | (관리자) 상품 수정 |
| PATCH | /admin/product/{id}/status | (관리자) 판매 상태 ACTIVE/INACTIVE/DELETED |
| GET | /admin/category | (관리자) 카테고리 리스트 |
| POST | /admin/category | (관리자) 카테고리 추가 |
| PUT | /admin/category/{id} | (관리자) 이름 수정 |
| DELETE | /admin/category/{id} | (관리자) 비활성(deleted_at) |
