# /admin/category/{id}

개발순서: 3차
기능: 관리자_카테고리 페이지 (../%EA%B8%B0%EB%8A%A5%EC%A0%95%EC%9D%98%EC%84%9C/%EA%B4%80%EB%A6%AC%EC%9E%90_%EC%B9%B4%ED%85%8C%EA%B3%A0%EB%A6%AC%20%ED%8E%98%EC%9D%B4%EC%A7%80%20b1fbdfb849fa835fb69881cc4d460330.md)
기능설명:  비활성 처리 
도메인: admin/category
메서드: DELETE
사용자/관리자: 관리자

**하드 삭제 대신 비활성 처리**하여 기존 상품·주문 이력의 카테고리 참조를 보존. (DB상으로는 `category.deleted_at`에 삭제 시각을 기록하는 소프트 삭제로 구현하며, 응답의 `active`는 `deleted_at IS NULL`에서 파생)

### Response

---

**`200`** 비활성 처리 성공

```json
{
  "id": 5,
  "active": false,
  "message": "카테고리가 비활성화되었습니다"
}
```

`422` 연결된 상품이 있는 경우 (경고)

```json
{
  "error": {
    "code": "UNPROCESSABLE",
    "message": "이 카테고리에 12개의 상품이 연결되어 있습니다. 상품의 카테고리를 먼저 변경해주세요.",
    "product_count": 12
  }
}
```

**연결 상품 처리 옵션 (팀 결정 필요)**

- **옵션 1 (엄격)**: 연결 상품 있으면 무조건 차단 → 관리자가 상품별로 카테고리 재지정
- **옵션 2 (강제)**: `force=true` 쿼리로 강제 비활성 → 연결 상품들도 "미분류"로 자동 이동
- **옵션 3 (즉시)**: 그냥 비활성만, 상품은 그대로 참조 유지 (사용자 리스트에서만 안 보임)
