### 선행조건

- 관리자 로그인 상태여야 한다

### 트리거

- 관리자 > 카테고리 관리 메뉴 진입
- "카테고리 추가" 버튼 클릭 / 개별 항목의 수정·삭제 클릭

### 동작

[관리자 - 카테고리 관리](https://app.notion.com/p/390bdfb849fa80748edbefaef46cc459?pvs=21) 

- 등록된 카테고리 목록을 노출 순서대로 표시한다 (카테고리명, 연결된 상품 수)
- 카테고리 추가: 카테고리명을 입력하여 신규 등록한다
- 카테고리 수정: 기존 카테고리명을 변경한다
- 카테고리 삭제(비활성): 사용하지 않는 카테고리를 목록에서 내린다
- 노출 순서 변경: 드래그로 사용자 상품 페이지에 보여질 카테고리 순서를 조정한다
- 등록·수정·순서 변경 시 사용자 상품리스트페이지의 카테고리 필터에 즉시 반영한다

### 예외 / 제한사항

- 카테고리명 미입력 시 저장을 제한한다
- 중복된 카테고리명 등록을 차단한다
- 연결된 상품이 있는 카테고리 삭제 시도 시 경고를 표시한다 (해당 상품들의 카테고리 재지정 안내)
- 삭제는 하드 삭제 대신 '비활성' 상태로 처리하여, 기존 상품·주문 이력과의 연결을 보존한다

### 비고

- 상품 등록/수정 화면의 카테고리 드롭다운은 본 화면에서 관리하는 '활성' 카테고리를 소스로 한다
- 계층형(대분류/소분류) 구조 없이 단일 depth로 관리한다 (MVP)
- 개발 일정이 빠듯할 경우, 드래그 순서 변경 기능은 후순위로 미루고 추가·수정·비활성만 우선 구현 가능
