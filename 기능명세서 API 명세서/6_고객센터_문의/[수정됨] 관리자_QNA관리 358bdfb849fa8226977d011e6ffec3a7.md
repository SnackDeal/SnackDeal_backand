# 관리자_QNA관리 (수정됨)

사용자/관리자 구분: 관리자
연관화면: QNA관리
도메인: admin/qna
기능설명: 기존 관리자가 QNA관리 페이지에 진입한다
우선순위: 상
API : /admin/qna (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/admin%20qna%208eebdfb849fa82808394819edaacee09.md), /admin/qna/{id} (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/admin%20qna%20%7Bid%7D%20330bdfb849fa83eca158812fc0ff5fd6.md), /admin/qna/{id}/answer (../API%20%EC%A0%95%EC%9D%98%EC%84%9C/admin%20qna%20%7Bid%7D%20answer%2068cbdfb849fa82709a088166abdb2c82.md)

### 선행조건

- 관리자 로그인 상태여야 한다

### 트리거

- 고객센터 관리 메뉴 진입

### 동작

[관리자 - 문의 관리](https://app.notion.com/p/390bdfb849fa80d09ea0f94709e3e542?pvs=21) 

- 접수된 문의 목록을 최신순으로 표시한다
    - 컬럼: 접수일, 회원명, 문의 유형, 제목, AI 요약, 처리 상태
- AI 자동 요약(선택 시간남으면 할것)
    - 문의 내용을 1~2줄로 요약하여 목록에 함께 표시
    - 요약은 접수 시점에 비동기로 생성하며, 실패 시 원문으로 대체
- 처리 상태 필터 (대기 / 완료) — qna.is_answered(boolean) 기준, 별도 "처리중" 상태 없음
- 문의 상세에서 원문 확인 및 답변 작성 (qna_answer 등록, qna당 1개 UNIQUE)
- 답변 작성 후 is_answered=true로 자동 변경 (대기 → 완료)

### 예외 / 제한사항

- (선택)AI 요약 실패 시 원문 그대로 표시하고, 요약 없이도 처리가 가능해야 한다
- 필수 답변 내용 누락 시 답변 등록을 제한한다
