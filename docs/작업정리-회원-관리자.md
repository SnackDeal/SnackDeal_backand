# 작업 정리 — 회원 · 관리자(로그인/메인/회원관리)

> 담당: 옥주 / 기준: `프로젝트 기능/기능명세서 API 명세서/API 정의서/*.md`
> 모든 응답은 공통 래퍼 `CommonResponse<T>` 로 감싸고, 필드는 camelCase 로 프로젝트 규칙에 맞춤(명세의 snake_case는 예시).

## ✅ 체크리스트 (완료)

### 01. 회원 기능
- [x] 회원가입 — 기존 구현
- [x] **이메일 인증코드 발송** — `POST /member/email/send-code`
  - 명세: `member email send-code`
  - 보강: `409` 이미 가입된 이메일, `429` 60초 재발송 제한, 응답에 `expiresIn`(300) 추가
- [x] **이메일 인증코드 검증** — `POST /member/email/verify-code`
  - 명세: `member email verify-code`
  - 보강: 응답에 인증 토큰 + `expiresIn`(600) 추가
- [x] 로그인 — 기존 + **탈퇴(DELETED) 계정 로그인 차단(401)** 추가
- [x] 토큰 재발급 — 기존 구현
- [x] 로그아웃 — 기존 구현
- [x] **내 정보 조회** — `GET /member/me`
  - 명세: `member me` (1차)
  - 보강: 응답에 `lastLogin` 추가
- [x] **회원정보 수정** — `PATCH /member/me`
  - 명세: `member me` (2차)
  - 보강: 비밀번호 변경 시 **현재 비밀번호 검증(401)**, 부분 수정(phone/password)

### 09. 관리자 로그인/메인
- [x] 관리자 로그인 — `POST /admin/login` (기존, 역할 ADMIN 검증)
- [x] **관리자 대시보드** — `GET /admin/main`
  - 명세: `admin main`
  - 구현: 오늘 주문 수 / 오늘 매출(취소·환불 제외) / 신규 회원 수 / 저재고 상품 수 / 미답변 QnA 수 집계
  - 설계: 다른 도메인 리포지토리를 수정하지 않도록 `DashboardService` 안에서 `EntityManager`(JPQL)로 직접 집계

### 10. 관리자 회원 관리
- [x] **회원 리스트 조회** — `GET /admin/members`
  - 명세: `admin members`
  - 구현: `keyword`(이메일/이름 부분검색) + `status` 필터 + 페이징
- [x] **회원 상태 조회** — `GET /admin/members/{id}`
  - 명세: `admin members {id}` (기본 정보 최소 스펙 채택 — 명세에서 허용)
- [x] **회원 상태 변경** — `PATCH /admin/members/{id}/status`
  - 명세: `admin members {id} status`
  - 구현: `reason` 필드, `403` 본인 계정 차단, `422` 탈퇴 회원 전이 불가, DELETED 시 토큰(세션) 즉시 무효화

## 📋 API 목록 (상세)

> 모든 응답은 공통 래퍼로 감싸진다. 아래 표의 **응답**은 `data` 안에 담기는 payload 만 표기.
> ```json
> { "success": true, "code": "S000", "message": "성공", "data": { ... } }
> ```
> 🔒 = 인증 필요(로그인), 👑 = 관리자(ROLE_ADMIN) 필요

### 01. 회원

| 메서드 | 경로 | 설명 |
|---|---|---|
| POST | `/member/email/send-code` | 이메일 인증코드 발송 |
| POST | `/member/email/verify-code` | 인증코드 검증 → 인증 토큰 발급 |
| POST | `/member/join` | 회원가입 |
| POST | `/member/login` | 로그인 |
| POST | `/member/token/refresh` | 토큰 재발급 |
| POST | `/member/logout` 🔒 | 로그아웃 |
| GET | `/member/me` 🔒 | 내 정보 조회 |
| PATCH | `/member/me` 🔒 | 내 정보 수정 |

**POST `/member/email/send-code`**
```json
// 요청
{ "email": "test@test.com" }
// 응답 data
{ "expiresIn": 300 }
// 오류: 400 형식오류 · 409 이미 가입된 이메일 · 429 60초 내 재발송
```

**POST `/member/email/verify-code`**
```json
// 요청
{ "email": "test@test.com", "code": "482913" }
// 응답 data
{ "verificationToken": "evt_...", "expiresIn": 600 }
// 오류: 400 코드 불일치/만료
```

**POST `/member/join`**
```json
// 요청
{ "email": "test@test.com", "password": "p@ssW0rd!", "name": "홍길동",
  "birth": "2000-01-01", "gender": "MALE", "phone": "01011112222",
  "verificationToken": "evt_..." }
// 응답 data: MemberDescription (아래 GET /member/me 와 동일 형태)
// 오류: 400 인증토큰 무효 · 409 중복 이메일
```

**POST `/member/login`**
```json
// 요청
{ "email": "test@test.com", "password": "p@ssW0rd!" }
// 응답 data
{ "accessToken": "eyJ...", "refreshToken": "eyJ..." }
// 오류: 401 미존재/비번오류 · 401 탈퇴 계정(ACCOUNT_DELETED)
```

**POST `/member/token/refresh`**
```json
// 요청
{ "refreshToken": "eyJ..." }
// 응답 data: { "accessToken": "...", "refreshToken": "..." }
// 오류: 401 유효하지 않음/만료/불일치
```

**POST `/member/logout`** 🔒 — 요청 바디 없음, 응답 `data: null`

**GET `/member/me`** 🔒
```json
// 응답 data
{ "id": 1, "email": "test@test.com", "name": "홍길동", "phone": "01011112222",
  "birth": "2000-01-01", "gender": "MALE", "status": "ACTIVE", "role": "USER",
  "createdAt": "2026-07-01T00:00:00", "lastLogin": "2026-07-01T14:32:00" }
```

**PATCH `/member/me`** 🔒
```json
// 요청 (모두 선택. password 변경 시 currentPassword 필수)
{ "phone": "01099998888", "currentPassword": "oldP@ss1!", "password": "newP@ss1!" }
// 응답 data: MemberDescription
// 오류: 401 현재 비밀번호 불일치 · 400 유효성 실패
```

### 09. 관리자 로그인/메인

| 메서드 | 경로 | 설명 |
|---|---|---|
| POST | `/admin/login` | 관리자 로그인 |
| GET | `/admin/main` 👑 | 대시보드 요약 지표 |

**POST `/admin/login`**
```json
// 요청
{ "email": "admin@test.com", "password": "p@ssW0rd!" }
// 응답 data: { "accessToken": "...", "refreshToken": "..." }
// 오류: 403 관리자 아님(FORBIDDEN_ACCESS)
```

**GET `/admin/main`** 👑
```json
// 응답 data
{ "todayOrderCount": 12, "todaySalesAmount": 340000, "newMemberCount": 5,
  "lowStockProductCount": 3, "pendingQnaCount": 6 }
```

### 10. 관리자 회원 관리

| 메서드 | 경로 | 설명 |
|---|---|---|
| GET | `/admin/members` 👑 | 회원 리스트(검색/필터/페이징) |
| GET | `/admin/members/{id}` 👑 | 회원 상세 |
| PATCH | `/admin/members/{id}/status` 👑 | 회원 상태 변경 |

**GET `/admin/members`** 👑
```
쿼리: keyword(이메일/이름, 선택) · status(ACTIVE|INACTIVE|DELETED, 선택) · page(기본 0) · size(기본 10)
응답 data: Page<MemberDescription>  (content[] + 페이지 정보)
```

**GET `/admin/members/{id}`** 👑 — 응답 `data: MemberDescription` · 오류 `404`

**PATCH `/admin/members/{id}/status`** 👑
```json
// 요청
{ "status": "INACTIVE", "reason": "6개월 미접속" }
// 응답 data
{ "id": 45, "email": "hong@test.com", "status": "INACTIVE", "updatedAt": "2026-07-02T10:00:00" }
// 오류: 403 본인 계정 · 422 탈퇴 회원 전이 불가 · 404 미존재
```

## 🛠 트러블슈팅

### 1. PowerShell `.ps1` 한글 깨짐
- **증상**: `The string is missing the terminator` — 스크립트 실행 시 한글 주석에서 파싱 오류
- **원인**: 파일이 BOM 없는 UTF-8로 저장됨. Windows PowerShell 5.1은 BOM이 없으면 cp949로 읽어 한글이 깨짐
- **해결**: `.ps1`을 **UTF-8 with BOM**으로 저장

### 2. 테스트 65개 컨텍스트 로드 실패 (다른 PC에선 정상)
- **증상**: `FileNotFoundException: class path resource [test-config.properties] cannot be opened`
- **원인**: 모든 `@SpringBootTest`가 `@TestPropertySource(classpath:test-config.properties)`로 이 파일을 참조하는데, `.gitignore`의 `src/test/resources/*.properties` 규칙 때문에 **커밋되지 않는 로컬 파일**이었음 → 새 PC엔 없음
- **해결**: 로컬에 `src/test/resources/test-config.properties` 생성 (H2/더미 설정). 커밋 안 되므로 팀에 영향 없음
- **주의**: clone한 새 PC마다 이 파일을 로컬로 만들어야 함

### 3. Redis 플레이스홀더 미해결
- **증상**: `Could not resolve placeholder 'REDIS_HOST'` → `redisConfig` 빈 생성 실패로 컨텍스트 로드 실패
- **원인**: `RedisConfig`가 `@Value("${REDIS_HOST}")` 등 환경변수식 값을 직접 읽는데, 테스트는 `test` 프로파일만 타서 `application-redis.yml`이 안 붙음
- **해결**: `test-config.properties`에 `REDIS_HOST / REDIS_PORT / REDIS_PASSWORD` 추가

### 4. DashboardServiceTest `em` NullPointerException
- **증상**: `Cannot invoke "EntityManager.createQuery(...)" because "...em" is null`
- **원인**: `@InjectMocks`가 생성자 주입(`memberRepository`)을 사용하면 Mockito는 **필드 주입을 추가로 수행하지 않음** → `@PersistenceContext EntityManager em` 필드가 주입되지 않고 null
- **해결**: `@BeforeEach`에서 `ReflectionTestUtils.setField(dashboardService, "em", em)`로 목을 수동 주입

### 5. 대시보드 집계 시 남의 도메인 침범
- **증상**: 초기 구현에서 order/product/cs 리포지토리에 count 메서드를 추가 → 다른 팀원 작업 파일 수정
- **해결**: 해당 리포지토리 원복 후, `DashboardService` 내부에서 `EntityManager` JPQL로 직접 집계하여 **내 파일만** 수정하도록 변경

### 6. Mockito strict 스터빙 예외 (예방)
- **증상 예상**: 파라미터 없는 QnA 집계 쿼리에서 `setParameter` 스텁이 미사용 → `UnnecessaryStubbingException`
- **해결**: 해당 스텁을 `lenient()` 처리
